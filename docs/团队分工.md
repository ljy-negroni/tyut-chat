# 团队分工文档

> 项目：校园即时通信与文件传输系统（tyut-chat）
> 对应课程设计题目：3.2 校园即时通信与文件传输系统

## 一、项目概述

本项目依据课程设计题目 3.2，设计并实现一个面向校园局域网 / 本机多客户端模拟场景的即时通信与文件传输系统。系统采用 C/S 架构，支持用户登录、在线用户查看、点对点文本聊天、群聊、文件传输、历史聊天记录等核心功能，并扩展实现了离线消息、消息已读未读、音视频通话等能力。

技术选型：后端 Spring Boot 3.3 + Netty + MyBatis-Plus，前端 Vue 2 + Element UI，存储 MySQL + Redis + MinIO。

## 三、项目架构与模块

后端采用 Maven 多模块，职责分层；前端为独立 Web 工程。

| 模块 | 职责 | 负责人 |
|---|---|---|
| chat-common | 公共包：通用模型、枚举、JWT、Redis 键定义，供各后端模块依赖 | 李俊毅 |
| chat-client | 消息推送 SDK：封装与推送服务的通信，供业务服务集成 | 李俊毅 |
| chat-server | 消息推送服务（Netty + WebSocket）：维护长连接、消息转发、在线状态 | 李俊毅 |
| chat-platform | 业务平台服务（HTTP）：用户、好友、群组、消息、文件等业务逻辑与数据持久化 | 李俊毅 |
| chat-web | Web 前端：登录、会话、私聊 / 群聊、文件传输等界面 | 王炳晟 |
| db | 数据库建表脚本与版本化更新脚本 | 李俊毅 |

## 六、协作机制

- **接口约定**：后端（李俊毅）先行定义 REST 接口与 WebSocket 消息协议，前端（王炳晟）据此联调；接口变更须同步知会双方。
- **版本管理**：统一在 GitHub 仓库协作，提交遵循项目规约（一次提交只做一件事、说明清晰）。
- **进度协调**：项目负责人（李俊毅）统筹里程碑与任务对齐；产品（韩钦伍）跟踪需求完成度并组织验收。
