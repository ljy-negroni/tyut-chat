# 项目计划书

> 本计划书记录 tyut-chat 的待办事项、已知问题与后续规划，是项目推进的依据（对应项目规约 §七）。
> 所有改动与新增须先登记本计划书；大规模 / 跨模块任务须写明拆解并经审查后动工。

## 一、当前状态

- 仓库已初始化，模块结构（chat-common / chat-client / chat-server / chat-platform / chat-web）搭建完成。
- 配置已本地化：数据库与 Redis 连接参数改用环境变量占位，真实凭证不进仓库（见 §十）。
- 团队分工已明确，见 `docs/团队分工.md`。
- 课设要求已归档，见 `docs/课设要求-校园即时通信与文件传输系统.md`。

| P1 | MinIO domain 本地化 | `domain` 写死原作者局域网 IP `192.168.2.67:9000`，本地访问上传的图片 / 文件会指向错误主机而 404，需改为本地回环（`127.0.0.1`）或本机局域网 IP | `chat-platform/src/main/resources/application-dev.yml` | 待办 |

## 三、后续工作流程

按以下顺序推进（待本地环境就绪后启动）：

1. **本地化测试检验**：启动 MySQL、Redis、MinIO，本地跑通 chat-platform + chat-server + chat-web，验证登录、私聊 / 群聊、文件传输等基础链路。
2. **确定需求文档**：对照课设要求（基本需求 11 项 + 扩展需求 10 项）逐项核对实现情况，产出需求文档（已实现 / 待补 / 不做）。
3. **编写详细计划**：基于需求文档拆解任务、分配分工、排定里程碑，作为后续开发依据。
