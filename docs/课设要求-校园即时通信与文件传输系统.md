## 3.2 校园即时通信与文件传输系统

### 3.2.1问题描述

即时通信系统是网络应用开发中的典型项目，涉及客户端/服务器架构、用户管理、消息转发、数据存储、文件传输和异常处理等内容。请设计并实现一个面向校园局域网或本机多客户端模拟的即时通信与文件传输系统。

系统应采用客户端/服务器结构。用户登录后，可以查看在线用户，进行点对点文本聊天和群聊，发送文件，并查看历史聊天记录。系统应能显示文件传输进度，并能够处理用户退出、连接失败等基3.2.2 基本需求

本题目系统架构不限，编程语言、开发框架、数据库、通信协议和界面技术不限。项目组可根据自身基础选择 Java Socket、Python Socket、WebSocket、Spring Boot、Node.js 或其他技术方案。

必须完成以下功能：

1）采用C/S架构，至少包含一个服务器端和一个客户端程序。

2）支持用户注册或预置用户登录

3）服务器维护在线用户列表

4）客户端能够查看在线用户

5）支持点对点文本聊天

6）支持至少一个群聊或公共聊天室

7）聊天记录保存到文件或数据库

8）用户可查看历史聊天记录

9）支持文件传输

10）用户退出或异常断开时，服务器能够更新在线状态，系统不应直接崩溃

11）提交README说明如何启动服务器、启动多个客户端、完成聊天和文件传输测试

### 3.2.3 扩展需求

完成基本要求后，可选择实现以下扩展功能：

1）离线消息

2）好友列表或联系人备注

3）多个群聊房间

4）文件传输过程中显示传输进度

5）文件传输断点续传

6）简单数据包加密

7）消息已读/未读状态

8）断线重连

9）管理员广播消息

10）聊天记录按关键词搜索
