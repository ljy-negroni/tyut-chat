# TYUT CHAT 项目亮点介绍

> 课设题目：3.2 校园即时通信与文件传输系统
> 项目定位：面向太原理工大学师生的校园即时通讯 Web 平台

---

## 一、架构设计亮点

### 1. 四模块微服务化后端架构

区别于传统单体应用，本项目将后端拆分为四个职责清晰的 Maven 模块，严格遵循单向依赖：

```
chat-common（公共包）→ chat-client（推送SDK）→ chat-platform（业务平台）
chat-common（公共包）→ chat-server（推送服务）
```

**亮点：** 推送服务（Netty WebSocket）与业务服务（Spring Boot REST API）完全解耦。两者通过 Redis 队列异步通信，可独立部署、独立扩缩容。platform 宕机不影响已建立的长连接，server 重启不影响业务数据完整性。

### 2. 基于 Redis List 的跨节点消息路由

这是本项目最具创新性的技术方案。当系统部署多台 chat-server 实例时，不同用户可能连接在不同节点上，如何将消息精准送达目标用户？

**解决方案：**
- 用户上线时，server 在 Redis 中记录映射：`im:user:server_id:{userId}:{terminal} = {serverId}`
- platform 发送消息前，查询目标用户所在 server，将消息 RPUSH 到对应队列：`im:message:private:{serverId}`
- 各 server 只 LPOP 消费自己的队列，互不干扰

**优势：** platform 无需持有任何 WebSocket 连接，保持无状态；server 节点增减对 platform 完全透明；单机故障不影响其他节点的消息推送。

## 二、实时通信亮点

### 3. Netty NIO 高性能 WebSocket 服务

放弃 Spring 自带的 WebSocket（BIO 模型，每连接一线程），自建 Netty 服务端：

- **主从 Reactor 线程模型**：BossGroup（Accept 连接）+ WorkerGroup（读写 I/O）+ 业务线程池（耗时操作）
- **单机万级并发**：基于 Selector 多路复用，一个 EventLoop 管理数千个 Channel
- **自定义 JSON 帧协议**：cmd 字段区分消息类型（心跳/登录/私聊/群聊/ACK/已读回执），统一编解码
- **IdleStateHandler 心跳检测**：60 秒读空闲自动断连，防止僵死连接占用资源

### 4. 完整的消息生命周期管理

每条消息经历精确的状态流转：

```
SENDING(−1) → PENDING(0) → DELIVERED(1) → READED(3)
                                    ↘ RECALL(2)（2 分钟内撤回）
```

- **先持久化再推送**：消息落库 MySQL 后才入 Redis 队列，保证不丢失
- **ACK 确认机制**：接收方收到推送后回复确认帧，服务端更新送达状态
- **已读批量上报**：用户进入会话时，批量标记该会话所有历史消息为已读
- **双层状态**：后端维护服务端状态（PENDING/DELIVERED/READED），前端额外维护本地状态（SENDING/FAILED），用户感知完整的"发送中→已送达→已读"

## 三、安全与认证亮点

### 5. JWT 双 Token 无状态认证

- **accessToken**（30 分钟有效）：存 sessionStorage，每次请求携带，用于业务鉴权
- **refreshToken**（7 天有效）：存 localStorage（可选），仅用于续期接口
- **自动续期**：前端 axios 拦截器捕获 401 → 自动调用 refreshToken → 重放原请求，用户无感知
- **不同密钥签名**：两个 Token 使用不同 secret，互不通用

### 6. 图形验证码防刷系统

- Hutool 生成 4 位数字线段干扰验证码（120×42 px）
- Redis 存储（`captcha:{uuid}` → code），5 分钟自动过期
- 一次性使用：校验通过立即 DEL key，防止同一验证码被重复利用
- 覆盖场景：注册必填 + 登录必填 + 点击刷新，全面防御机器人攻击

### 7. 多层安全防护

| 层面 | 措施 |
|------|------|
| 密码安全 | BCrypt（$2a$10$，1024 轮哈希 + 自动加盐） |
| 接口安全 | AuthInterceptor 全局拦截 + 白名单机制 |
| 输入安全 | XSS 过滤拦截器 + SQL 参数化查询 |
| 内容安全 | 自定义 Trie 前缀树敏感词过滤（60 秒动态刷新） |
| 密钥安全 | 所有敏感配置（DB密码、JWT Secret、MinIO Key）使用环境变量占位 |

## 四、存储与文件亮点

### 8. MinIO 自建对象存储

- 图片上传自动生成缩略图（Thumbnailator，宽 300px），减小前端带宽消耗
- Bucket 公开读策略，Client 可直接通过 URL 访问
- 支持分布式部署，S3 兼容 API，后续可无缝迁移云 OSS
- 文件过期自动清理（定时任务扫描）

### 9. 多态消息体 JSON 存储

消息内容采用统一 JSON 格式存储，而非多字段设计：
- 文字：`"你好"`
- 图片：`{"originUrl":"...", "thumbUrl":"...", "width":1280, "height":1792}`
- 文件：`{"name":"报告.pdf", "url":"...", "size":2048000}`
- 语音：`{"url":"...", "duration":15}`

**优势：** 新增消息类型无需 ALTER TABLE，前后端 JSON 直通零转换，彻底解耦消息结构与数据库 Schema。

## 五、前端设计亮点

### 10. macOS Vibrancy 暗色设计系统

从 Apple 桌面应用设计语言中提炼的 Web 还原，核心哲学是"克制"：

- **层级即色彩**：不用渐变/发光区分层级，纯靠背景深浅（`#0a0a0c` → `#1c1c1e` → `#2c2c2e` → `#3a3a3c`）
- **系统级模糊**：`backdrop-filter: blur(60px)` 模拟 NSVisualEffectView 毛玻璃效果
- **1px 哲学**：所有分隔均为 1px 半透明边框，不用阴影制造深度
- **无装饰主义**：按钮纯色无渐变，无 pulse/bounce 等装饰动画

### 11. Three.js 3D 粒子背景

登录/注册页独立编写 150+ 行 WebGL 代码：
- 12 个多面体轨道浮动（Icosahedron / Octahedron / Tetrahedron / TorusKnot）
- 800+ 粒子构成 4 层光环（BufferGeometry + AdditiveBlending）
- 5 层自定义 Shader 极光丝带（vertexShader + fragmentShader）
- 鼠标跟随相机平滑插值 + `requestAnimationFrame` 动画循环
- 全部灰白暗色调，与 macOS 风格统一

### 12. 工程化前端架构

- **Pinia 5 模块状态管理**：userStore / chatStore / friendStore / groupStore / configStore，职责清晰
- **Axios 三层拦截器**：请求自动带 Token → 响应自动解包 → 401 自动续期重放
- **虚拟滚动（VirtualScroller）**：会话列表只渲染可视区，上千会话不卡顿
- **WebSocket 自动重连**：离线后 15 秒自动重连 + 重连后批量拉取离线消息
- **Token 分层存储**：accessToken（sessionStorage，关闭即失效）+ refreshToken（localStorage，持久化自动登录）
